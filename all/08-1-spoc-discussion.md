实现信号量的设计方案：

维护一个全局列表semInfoList，其成员为如下结构：

```
struct SemList
{
    uint32 proc;     //存储进程号
    uint32 semnum;   //存储信号标识符
    auto address;    //存储相应处理程序的内存地址
};
```

在系统调用的处理函数中加入对信号的识别与处理，每次运行signal函数时在内核处理函数中将以上三个变量存入列表中。当处理函数收到一个信号时，根据当前进程号和该信号的标识符在semInfoList中查询（可对该全局列表用树结构组织或增加排序处理，以降低查询复杂度），找到相应的处理程序所在的内存地址，然后通过return和函数调用的方式执行处理程序。处理程序结束后，返回原程序中断处继续执行。
